## yaml语法规则
- 大小写敏感
- 属性层级关系使用多于描述，每行结尾使用冒号结束
- 使用缩进表示层级关系，同层级左侧对齐，只允许使用空格（不允许使用Tab键）
- 属性前面加空格（属性名与属性值之间使用冒号+空格作为分隔）
- \#表示注释
- 数组数据在数据书写位置的下方使用减号作为数据的起始符号，每行书写一个数据，减号与数据之间用空格隔开
```yaml
enterprise:
	name: slayerlike
	age: 21
	tel: 110
	likes:
		- java
		- go
		- games
```
